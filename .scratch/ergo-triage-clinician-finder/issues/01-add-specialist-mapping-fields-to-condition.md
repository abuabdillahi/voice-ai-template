# Issue 01: Add specialist-mapping fields to the `Condition` dataclass

Status: ready-for-agent

## Parent

`.scratch/ergo-triage-clinician-finder/PRD.md`

## What to build

Extend `core.conditions.Condition` with two new fields that capture, per condition, what kind of clinician the user should be referred to and how to find them in OpenStreetMap. The fields are referral metadata — they do not bleed into the symptom-interview system prompt — and they live on the `Condition` dataclass alongside the existing curated medical content so adding a sixth condition stays a single record-append. No consumer of the new fields exists yet; this slice is data + tests only and is mergeable independently.

The two fields:

- `specialist_label: str` — what the agent verbalises ("physiotherapist", "optometrist", "general practitioner"). Used in the offer language ("Would you like me to find a physiotherapist near you?") and in the post-result spoken summary.
- `specialist_osm_filters: tuple[str, ...]` — Overpass tag filters of the form `key=value`, OR-joined into one Overpass query and deduplicated by node id at query time. The OR-joined filter list reflects OSM's two competing tagging schemas (older `amenity=*` and newer `healthcare=*`); real-world coverage is split across both, and querying only one misses POIs tagged under the other.

The five mappings, populated verbatim:

| Condition | `specialist_label` | `specialist_osm_filters` |
|---|---|---|
| `carpal_tunnel` | `"physiotherapist or occupational therapist"` | `("healthcare=physiotherapist", "healthcare=occupational_therapist")` |
| `computer_vision_syndrome` | `"optometrist"` | `("healthcare=optometrist", "shop=optician")` |
| `tension_type_headache` | `"general practitioner"` | `("amenity=doctors", "healthcare=doctor", "healthcare=general_practitioner")` |
| `upper_trapezius_strain` | `"physiotherapist"` | `("healthcare=physiotherapist",)` |
| `lumbar_strain` | `"physiotherapist"` | `("healthcare=physiotherapist",)` |

The byte-identical `kb_for_prompt()` snapshot is the regression anchor. The new fields must NOT be rendered into the prompt block — referral metadata is for the tool path, not the symptom-interview prompt.
