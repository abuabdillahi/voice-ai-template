## What to build

Bootstrap the monorepo with workspace configuration for both ecosystems and a task runner that can orchestrate cross-language commands. After this slice, contributors can clone the repo, install dependencies once with each ecosystem's tool, and run a no-op task across all packages to verify wiring.

The directory structure follows the layout established in the PRD:

```
voice-ai/
├── apps/
│   ├── web/
│   ├── api/
│   └── agent/
└── packages/
    └── core/
```

Each app and package gets a stub manifest (empty `pyproject.toml` for Python packages, empty `package.json` for the web app), enough to register as a workspace member. No application code yet.

## Acceptance criteria

- [ ] `pyproject.toml` at the repo root declares a `[tool.uv.workspace]` with `apps/api`, `apps/agent`, `packages/core` as members
- [ ] `pnpm-workspace.yaml` at the repo root declares `apps/web` as a member
- [ ] Each Python package has a minimal `pyproject.toml` with name, version, and python requirement
- [ ] `apps/api` and `apps/agent` declare `core` as a workspace dependency
- [ ] `apps/web` has a minimal `package.json` with name and a noop `dev` script
- [ ] `package.json` at the repo root configures pnpm and registers `turbo` as a dev dependency
- [ ] `turbo.json` at the repo root defines `dev`, `build`, `lint`, `test`, `typecheck` pipelines (most as no-ops at this stage)
- [ ] `pnpm install` succeeds at the repo root
- [ ] `uv sync` succeeds at the repo root
- [ ] `turbo run lint` exits 0 across all packages (lint commands are no-ops here; real configuration arrives in issue 02)
- [ ] `.gitignore` covers Python (`__pycache__`, `.venv`, `*.pyc`), Node (`node_modules`, `.turbo`), env files (`.env`, `.env.local`), build outputs (`dist`, `build`), and IDE/OS files
- [ ] README at the repo root contains a "Getting started" section listing prerequisites (uv, pnpm, Node) and the install commands
