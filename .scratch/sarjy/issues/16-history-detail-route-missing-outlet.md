## What to build

Clicking a row on `/history` updates the URL to `/history/<id>` but the detail page never renders â€” only the list view stays visible.

Root cause: with TanStack Router flat-file naming, `apps/web/src/routes/history.$id.tsx` is treated as a **child** of `apps/web/src/routes/history.tsx`. See `apps/web/src/routeTree.gen.ts:32-36`:

```ts
const HistoryIdRoute = HistoryIdRouteImport.update({
  id: '/$id',
  path: '/$id',
  getParentRoute: () => HistoryRoute,
} as any)
```

For a child route to render, the parent route's component must include an `<Outlet />`. `HistoryRoute()` in `history.tsx` renders only its own header + `<HistoryList />` and does not include `<Outlet />`. So the child route matches but cannot mount.

Each page in this app already provides its own header and layout (compare with `index.tsx`, `sign-in.tsx`), so the cleanest fix is to break the parent/child nesting rather than restructure both files around a shared `<Outlet />`. TanStack Router's flat-routing convention for opting out of layout nesting is the trailing-underscore segment: rename `history.$id.tsx` â†’ `history_.$id.tsx` and the detail route becomes a top-level sibling of the list route.
