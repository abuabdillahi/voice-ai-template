# Issue 09: Conversation persistence tracer

Status: ready-for-agent
Category: enhancement

## Parent

`.scratch/voice-ai-template/PRD.md`

## What to build

Persist conversation transcripts as structured rows so the user can browse their history and re-read what was said. After this slice, every voice conversation produces a `conversations` row with start/end timestamps and an optional LLM-generated summary, plus a `messages` row per turn (user, assistant, tool). The web app gains a `/history` route listing past conversations and a `/history/:id` route showing the full transcript with timestamps.

Independent of the memory chain (issues 07 and 08) — can be implemented in parallel with them once issue 05 lands.

## Acceptance criteria

### Schema

- [ ] Migration creates:
  - `conversations (id uuid PRIMARY KEY, user_id uuid REFERENCES auth.users, started_at timestamptz, ended_at timestamptz NULL, summary text NULL, metadata jsonb DEFAULT '{}')`
  - `messages (id uuid PRIMARY KEY, conversation_id uuid REFERENCES conversations ON DELETE CASCADE, role text CHECK (role IN ('user', 'assistant', 'tool')), content text, tool_name text NULL, tool_args jsonb NULL, tool_result jsonb NULL, created_at timestamptz)`
- [ ] Indexes on `conversations(user_id, started_at DESC)` and `messages(conversation_id, created_at)`
- [ ] RLS on both tables: `auth.uid() = user_id` directly on `conversations`, and via join through `conversation_id` on `messages`

### `packages/core`

- [ ] `core.conversations.start(user) -> ConversationId` inserts a new row and returns the id
- [ ] `core.conversations.append_message(conv_id, role, content, tool_name=None, tool_args=None, tool_result=None)` inserts a message; ordering is enforced by `created_at` (server-set)
- [ ] `core.conversations.end(conv_id, summary: str | None = None)` sets `ended_at` and optional summary
- [ ] `core.conversations.list_for_user(user, limit=50, offset=0) -> list[ConversationSummary]` returns id, started_at, ended_at, summary, message_count
- [ ] `core.conversations.get(user, conv_id) -> Conversation` returns the full conversation with its messages (ordered)
- [ ] `core.conversations.generate_summary(conv_id)` calls the LLM with the message history and returns a short summary; called from `end` when `summary` is None and message count ≥ 3

### `apps/agent`

- [ ] On session start, calls `core.conversations.start` and binds the resulting id to the session context
- [ ] Hooks `on_user_speech_committed` → appends a `user` message
- [ ] Hooks `on_agent_speech_committed` → appends an `assistant` message
- [ ] Hooks `on_function_calls_finished` → appends one `tool` message per call with name/args/result
- [ ] On session end (room close or participant leave), calls `core.conversations.end` with auto-summary
- [ ] `conversation_id` is bound to the structlog context for every log line emitted during the session

### `apps/api`

- [ ] `GET /conversations` returns the user's conversations (paginated)
- [ ] `GET /conversations/:id` returns a single conversation with its full message list, 404 if not owned by the user (RLS enforces this; route just maps the empty result)

### `apps/web`

- [ ] `/history` route: list view (shadcn `Table` or `Card` list) showing started_at, summary preview, message count
- [ ] `/history/:id` route: transcript view with role-styled message bubbles and timestamps
- [ ] Sidebar nav adds a "History" link from the talk page

### Tests

- [ ] `core.conversations` unit tests for each function
- [ ] `core.conversations` integration tests against testcontainers Postgres:
  - Append-then-list returns messages in order
  - End triggers summary generation when threshold met (with a mocked LLM)
  - User isolation via RLS
- [ ] `apps/api` route tests for `/conversations` and `/conversations/:id`
- [ ] `apps/agent` session test extends the harness to assert that scripted user/assistant turns produce the expected `messages` rows
- [ ] `apps/web` component tests for the history list and the transcript view

### Demo

- [ ] Clicking the row opens `/history/:id` and shows the full transcript in order
