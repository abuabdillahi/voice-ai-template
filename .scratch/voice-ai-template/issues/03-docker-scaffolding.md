## What to build

Containerize each app and provide both a development and a production compose file. After this slice, `docker compose build` succeeds for all three services, `docker compose up` brings up the dev stack with stubs that respond to a healthcheck, and CI verifies all three Dockerfiles still build on every PR. The web service in production is served as static assets through nginx; in development it runs outside compose via Vite for HMR.

LiveKit is not yet wired (that arrives in issue 05) — this slice establishes only the containerization shape.

## Acceptance criteria

- [ ] `apps/api/Dockerfile` is multi-stage (uv-based builder → `python:3.13-slim` runtime), runs as non-root, exposes port 8000, has a `HEALTHCHECK` hitting `/health`
- [ ] `apps/agent/Dockerfile` is multi-stage (uv-based builder → `python:3.13-slim` runtime), runs as non-root, no exposed ports (dial-out only)
- [ ] `apps/web/Dockerfile` is multi-stage (pnpm-based builder running `pnpm build` → `nginx:alpine` runtime serving `dist/`)
- [ ] `apps/api` has a `/health` endpoint returning `{"status": "ok"}`; `apps/agent` has a stub entrypoint that logs a startup line and sleeps (real worker arrives in issue 05)
- [ ] `docker-compose.yml` at the repo root brings up `api` and `agent` services with `.env`-loaded configuration; web is excluded from dev compose with a comment explaining Vite runs outside
- [ ] `docker-compose.prod.yml` at the repo root adds the `web` (nginx) service and a placeholder for `livekit/livekit-server` (commented out until issue 05 wires it)
- [ ] `.dockerignore` files exclude `node_modules`, `.venv`, `dist`, `build`, `.git`, env files
- [ ] `.env.example` at the repo root lists all variables expected so far (Supabase URL/keys placeholders, future LiveKit/OpenAI placeholders are commented)
- [ ] `docker compose build` succeeds for all services on a clean clone
- [ ] `docker compose up` brings api and agent to a running state; api's `/health` returns 200 within 30s
- [ ] `.github/workflows/ci.yml` gains a `build` job that runs `docker buildx build` for each app's Dockerfile (no push), gated to run after `lint`
- [ ] README documents the dev (`pnpm dev` for web, `docker compose up` for api/agent) and prod (`docker compose -f docker-compose.prod.yml up`) workflows
