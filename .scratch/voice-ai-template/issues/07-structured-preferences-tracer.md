## What to build

Persist named user preferences across sessions and devices. The agent gets `set_preference(key, value)` and `get_preference(key)` tools. The frontend gains a "what I remember about you" sidebar that lists current preferences. After this slice, telling the assistant "my favorite color is blue," signing out, signing back in, and asking "what's my favorite color?" produces the correct answer — the canonical demonstration of cross-session memory the PRD is built around.

This slice establishes the per-user RLS pattern that every subsequent user-scoped table reuses.

## Acceptance criteria

### Schema

- [ ] Migration creates `user_preferences (user_id uuid REFERENCES auth.users, key text, value jsonb, updated_at timestamptz, PRIMARY KEY (user_id, key))`
- [ ] RLS enabled on `user_preferences` with policies allowing `auth.uid() = user_id` for select/insert/update/delete
- [ ] Migration committed; `supabase db reset` reapplies cleanly

### `packages/core`

- [ ] `core.preferences.set(user, key, value)` upserts into `user_preferences` (`value` is any JSON-serializable Python value)
- [ ] `core.preferences.get(user, key) -> value | None` returns the stored value or None
- [ ] `core.preferences.list(user) -> dict[str, value]` returns all preferences for the user
- [ ] All three functions use the Supabase Python client with the user's JWT so RLS applies
- [ ] Two new tools registered in `core.tools.examples`:
  - `set_preference(key: str, value: str)` calls `core.preferences.set`
  - `get_preference(key: str) -> str | None` calls `core.preferences.get`
- [ ] System prompt updated to instruct the agent: "When the user states a preference (favorite color, preferred name, language), call `set_preference`. Before answering personal questions, consider calling `get_preference`."

### `apps/api`

- [ ] `GET /preferences` returns the authenticated user's preferences as a flat dict
- [ ] Route is a thin adapter over `core.preferences.list`

### `apps/web`

- [ ] "What I remember about you" sidebar component on the talk page (shadcn `Card`)
- [ ] Sidebar fetches `/preferences` via TanStack Query with a 10-second `refetchInterval` so newly-saved preferences appear during a live conversation
- [ ] Empty state shows a hint: "I'll remember things you tell me here."
- [ ] Each preference renders as a labelled row with the key and value
