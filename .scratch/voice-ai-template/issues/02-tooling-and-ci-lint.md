## What to build

Wire up the full code-hygiene pipeline so that contributors cannot commit unformatted, unlinted, or improperly-typed code, and so that the same checks run in CI. Two-tier hook split — fast formatters/linters/secret scan on commit, slower type-checkers and unit tests on push. Conventional-commit format enforced from the first commit.

After this slice, `pre-commit run --all-files` passes on a fresh clone, a deliberately broken commit message is rejected, and the CI lint job runs on pull requests.

## Acceptance criteria

- [ ] `pyproject.toml` at the repo root contains ruff and mypy configuration; ruff selects an opinionated rule set; mypy is set to strict
- [ ] `eslint.config.js` (flat config) at the repo root configures `@typescript-eslint`, `eslint-plugin-react`, `eslint-plugin-react-hooks`
- [ ] `.prettierrc` at the repo root configures 2-space indent, single quotes, trailing commas
- [ ] `.editorconfig` at the repo root pins indent style, line endings, charset
- [ ] `.pre-commit-config.yaml` at the repo root configures:
  - commit-stage hooks: `ruff format`, `ruff check --fix`, `prettier --write`, `eslint --fix`, trailing-whitespace, EOF newline, merge-conflict marker check, large-file check, `detect-secrets` (or `gitleaks`)
  - push-stage hooks: `mypy`, `tsc --noEmit`, fast unit tests (`pytest -m "not integration"` and `vitest run --no-coverage`)
  - commit-msg-stage hook: commitlint with conventional config
- [ ] `commitlint.config.js` at the repo root extends `@commitlint/config-conventional`, sets subject max length to 72
- [ ] `.secrets.baseline` is committed at the repo root
- [ ] Each Python package has lint/format/typecheck commands wired through Turbo; each TS package the same with eslint/prettier/tsc
- [ ] A deliberately bad commit message (e.g. `bad message`) is rejected by the commit-msg hook
- [ ] `.github/workflows/ci.yml` runs a `lint` job on PR and push to main that reuses the same commands (ruff, mypy, eslint, prettier --check, tsc)
- [ ] CI uses `actions/setup-uv@v3`, `actions/setup-node@v4` with pnpm caching, and Turborepo remote cache (or local cache for now)
- [ ] README documents `pre-commit install` as a one-time setup step
